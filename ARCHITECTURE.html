The two enforcement sockets — how artifacts reach the playground

① Invocation

Artifacts sit idle until something fires one. Transient, one-shot. This is where the dance lab's moves are enforced.

keystroke → CommandInput → useCommandRunner.submit → resolveCommandChain (data) → PlaygroundView.playMoves → useMovePlayer.playSequence

② Simulation

Artifacts are always present; the physics loop reconciles the character against them every frame. This is where the future world objects will be enforced.

useGameLoop tick → useCharacterPhysics integrates forces, clamps to ground/bounds → today: one hardcoded floor; tomorrow: a loop over collidable rectangles

Two game loops, summed in one place

useCharacterPhysicsowns WORLD POSITION. Tick: intent → velocity, +gravity, integrate, clamp. Output: x/y in world px.
＋
useMovePlayerowns ARTICULATION. Tick: tween pose, lerp rotation, drain dot-chain queue. Output: pose · rotation · (offset).
→
PlaygroundViewthe ONLY integration point. characterX = physics.x (+ offset); rotation wraps Character. Sums the two engines.
→
Playgroundcamera follows characterX, clamped to world edges; projects world→screen for both biome layers and the character.

Documented seams — where the structure bends (from the architecture review)

Sequencing lives inside the articulation engine. useMovePlayer.ts · queue + playSequence The dot-chain queue advances inside the player's physics-tick. "How do chains run?" is buried in a single-move file. Extract a useSequencer driven by an onComplete callback when a third engine joins — this also lets link-timing key off physics state instead of the player's clock.
One force law applied at two altitudes. useMovePlayer.ts (offset) + PlaygroundView.tsx (impulse) FORCE_STRENGTH_MULTIPLIER is multiplied in the engine (lab's cosmetic offset) and again in the view (real impulse), kept safe only by deliberately NOT passing forceVector to the player. Make it one forceToImpulse() in data/.
The playground's currentOffset term is always zero. PlaygroundView.tsx · characterX/Y Dead arithmetic that misleads a reader into thinking the move player contributes position in the playground. Bind placement to physics.position alone.
Ground is a hardcoded y=0 plane, compared three ways. useCharacterPhysics.ts · lines ~65 / ~93 / ~111 grounded uses ≥0, clamp uses >0, jump uses ≥0. No groundHeightAt(x) seam. This is the file the world-collision vision lands in — generalize the floor to a collision result before platforms arrive.
Library freshness is an invariant in a comment. PlaygroundView.tsx · useLocalStorage read Correctness relies on the router fully unmounting the editor before the playground mounts. A tabbed/modal layout would silently serve a stale move set. Needs a subscription or reload-on-enter when that day comes.

Color encodes layer. The triangle's one rule — arrows only point down — held across the whole review (no upward imports found). Every file sorts into defines an artifact, edits one, or enforces one; every artifact reaches the playground through socket ① (invoked) or ② (simulated).
